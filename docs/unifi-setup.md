# UniFi Setup Guide

Detailed instructions for configuring UniFi devices to send data to the monitoring platform.

## Overview

UniFi devices can export three types of data:

| Data Type | Protocol | Port | UniFi Requirement | Phase |
|-----------|----------|------|-------------------|-------|
| Syslog (CEF) | UDP | 514 | Network App 7.3.76+ | 1 |
| NetFlow/IPFIX | UDP | 2055 | UDM Pro/SE or USG (CLI) | 1 |
| API data | HTTPS | 443 (controller) | Local admin account | 2 |

## Syslog Configuration

UniFi exports system logs in **Common Event Format (CEF)**, an industry-standard format compatible with SIEM platforms. The CEF output is generated by the UniFi Network Application (controller), not the individual devices.

### Steps

1. Open the **UniFi Network** controller web interface
2. Navigate to **Settings** > **Control Plane** > **Integrations**
3. Find the **Activity Logging (Syslog)** section
4. Enable **SIEM Server**
5. Enter your monitoring server's IP address
6. Set the port to `514`
7. Protocol: **UDP**
8. Click **Apply Changes**

### What Gets Logged

- Admin activity (login, configuration changes)
- Client events (connect, disconnect, roam)
- Security detections (IDS/IPS alerts)
- Device status changes (adopt, disconnect, firmware update)
- Firewall rule matches (if per-rule logging is enabled)

### Enabling Firewall Rule Logging

By default, individual firewall rule matches are not logged. To enable:

1. Navigate to **Settings** > **CyberSecure** > **Policy Engine** > **Policy Table**
2. Edit the desired firewall rule
3. Enable **Syslog Logging** for that rule
4. Save and apply

### CEF Format Example

```
CEF:0|Ubiquiti|UniFi|1.0|2000|Firewall Rule Allow|3|src=192.168.1.100 dst=8.8.8.8 spt=54321 dpt=443 proto=TCP act=allow
```

Fields parsed by Vector:
- `deviceVendor` → Loki label `vendor`
- `deviceProduct` → Loki label `product`
- `severity` → Loki label `severity`
- `name` → stored as `event_name`
- CEF extensions: `src`, `dst`, `spt`, `dpt`, `proto`, `act`, `reason`

### Important Notes

- The CEF format is generated by the **controller application**, not individual devices
- USG/UDM gateways may also forward plain syslog (non-CEF) directly
- Log format can be inconsistent across different device types (APs vs switches vs gateways)
- Vector handles both CEF and plain syslog formats automatically

## NetFlow/IPFIX Configuration

### Modern UniFi Controllers (UDM Pro/SE, 7.x+)

1. Navigate to **Settings** > **System** > **Traffic Logging**
2. Enable **NetFlow (IPFIX)**
3. Enter your monitoring server's IP as the collector address
4. Set the port to `2055`
5. Select which VLANs/networks to monitor
6. Apply changes

### USG (Legacy Gateway) via CLI

SSH into the USG and run:

```bash
configure
set system flow-accounting interface eth0
set system flow-accounting netflow enable-ingress
set system flow-accounting netflow engine-id 1
set system flow-accounting netflow server <MONITORING_SERVER_IP> port 2055
set system flow-accounting netflow version 9
commit
save
```

Replace `eth0` with your WAN or LAN interface as needed. To monitor multiple interfaces:

```bash
set system flow-accounting interface eth0
set system flow-accounting interface eth1
```

### USG via config.gateway.json

For persistent configuration that survives reprovisioning, create/edit `config.gateway.json` on your controller:

```json
{
  "system": {
    "flow-accounting": {
      "interface": ["eth0"],
      "netflow": {
        "enable-ingress": "",
        "engine-id": "1",
        "server": {
          "<MONITORING_SERVER_IP>": {
            "port": "2055"
          }
        },
        "version": "9"
      }
    }
  }
}
```

Place this file in:
- **Cloud Key:** `/srv/unifi/data/sites/default/`
- **Self-hosted:** `<unifi_base>/data/sites/default/`

Then force a gateway reprovision from the controller UI.

### Verifying NetFlow is Working

SSH into the gateway and check that packets are being sent:

```bash
sudo tcpdump -i any udp port 2055 -c 5
```

You should see packets being sent to your monitoring server. On the monitoring server, check ClickHouse:

```bash
docker exec netmon-clickhouse clickhouse-client \
  --query "SELECT count() FROM netmon.flows"
```

### sFlow

GoFlow2 also listens on UDP 6343 for sFlow. UniFi switches have limited sFlow support. If your switches support it, configure the sFlow collector to point at `<monitoring-server>:6343`.

## API Access Configuration

### Creating a Local Admin Account

The analyzer connects to the UniFi controller's REST API. For security, create a dedicated read-only local account:

1. Open your UniFi controller
2. Navigate to **Settings** > **Admins & Users** > **Admins**
3. Click **Add Admin**
4. Select **Local Access Only** (not a UI.com account)
5. Set username (e.g., `netmon-readonly`) and a strong password
6. Set role to **View Only** if available, otherwise **Admin**
7. Save

### Configuring the Analyzer

Edit your `.env` file:

```bash
UNIFI_HOST=https://192.168.1.1     # Controller IP with protocol
UNIFI_USERNAME=netmon-readonly
UNIFI_PASSWORD=your-strong-password
UNIFI_SITE=default
UNIFI_VERIFY_SSL=false
```

**Notes:**
- `UNIFI_HOST` must include the protocol (`https://`)
- For UDM/UDM Pro, the controller runs on the gateway itself (e.g., `https://192.168.1.1`)
- For Cloud Key, use the Cloud Key's IP (e.g., `https://192.168.1.2`)
- For self-hosted, use the server's IP and port (e.g., `https://unifi.local:8443`)
- `UNIFI_VERIFY_SSL=false` is needed for self-signed certificates (default on UniFi hardware)

### API Endpoints Used

The analyzer accesses these UniFi API endpoints:

| Endpoint | Method | Data Retrieved |
|----------|--------|---------------|
| `/api/login` or `/api/auth/login` | POST | Authentication (cookie-based) |
| `/api/s/{site}/stat/device` | GET | All adopted devices (APs, switches, gateways) |
| `/api/s/{site}/stat/sta` | GET | All currently connected clients |
| `/api/s/{site}/stat/alarm` | GET | Recent alerts and events |
| `/api/s/{site}/stat/health` | GET | Site health summary |
| `/api/s/{site}/stat/sysinfo` | GET | Controller system information |

### Verifying API Access

After configuring and starting Phase 2:

```bash
# Check the analyzer can reach the controller
curl http://localhost:8080/api/unifi/devices

# Expected output (example):
{
  "devices": [
    {
      "mac": "aa:bb:cc:dd:ee:ff",
      "name": "Living Room AP",
      "model": "U6-LR",
      "type": "uap",
      "ip": "192.168.1.10",
      ...
    }
  ],
  "count": 5
}
```

If you see `{"error": "..."}`, check:
1. The controller is reachable from the Docker network
2. Credentials are correct
3. The account is a local admin (not a UI.com account)

### Device Compatibility

| Device | Syslog | NetFlow | API |
|--------|--------|---------|-----|
| UDM Pro | Yes (CEF) | Yes (IPFIX, native) | Yes |
| UDM SE | Yes (CEF) | Yes (IPFIX, native) | Yes |
| UDM | Yes (CEF) | Limited | Yes |
| USG | Yes (plain syslog) | Yes (CLI config) | Yes |
| Cloud Key Gen2+ | Yes (CEF, from controller) | N/A (controller only) | Yes |
| UniFi Switches | Via controller | sFlow (limited) | Via controller |
| UniFi APs | Via controller | N/A | Via controller |

**Note:** Individual APs and switches don't export syslog or NetFlow directly. Their events are reported through the controller, which aggregates and exports them.

## Troubleshooting UniFi Data

### No Syslog Data

1. Verify the controller version is 7.3.76+
2. Check the SIEM server configuration in the controller
3. Verify UDP port 514 is reachable: `nc -u -z <monitoring-server> 514`
4. Check Vector is listening: `docker logs netmon-vector | grep syslog`
5. Send a manual test: `./scripts/test-syslog.sh`

### No NetFlow Data

1. Verify your gateway supports NetFlow (UDM Pro/SE or USG with CLI config)
2. Check Traffic Logging is enabled in the controller
3. Verify UDP port 2055 is reachable: `nc -u -z <monitoring-server> 2055`
4. Check GoFlow2 is receiving: `docker logs netmon-goflow2`
5. Send synthetic data: `./scripts/test-netflow.sh`

### API Connection Failures

1. Test connectivity: `curl -k https://<controller-ip>/api/login`
2. Verify credentials work manually:
   ```bash
   curl -k -X POST https://<controller-ip>/api/login \
     -H "Content-Type: application/json" \
     -d '{"username":"netmon-readonly","password":"your-password"}'
   ```
3. Check the analyzer logs: `docker logs netmon-analyzer`
4. Ensure the account is **local only**, not a Ubiquiti cloud account
